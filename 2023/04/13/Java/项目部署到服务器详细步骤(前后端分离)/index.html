项目部署到服务器详细步骤(前后端分离)

第一次在服务器部署的时候遇到了很多坑，记录下防止忘记，本文使用若依框架来做演示。

一、拉取代码、打包

1、git拉取代码

在github或gitee上拉取代码的方式有两种，一种是代码下载成压缩包，将压缩包解压到指定的文件夹下，一种是使用命令，在存储代码的文件夹下打开Git Bash Here窗口，输入git clone xxx(代码地址)，这种方式我感觉要比下载压缩包方便很多，用idea打开此代码后会直接关联git的账号及分支，我们可以直接拉取或提交代码（拉取别人的代码需要修改提交地址）。而下载的需要在idea中连接git，有几次连接不上，我感觉不是很方便。

2、上传项目

修改项目信息，这个项目就是我们自己的项目了，需要把项目上传到我们自己的git仓库。首先要在gitee或github上创建仓库，生成仓库的地址https.xxxx。在idea中push项目是填入仓库地址就把项目提交到对应的仓库中了。

3、项目打包

后端使用maven打包

方法一：

在idea中使用maven插件进行打包，maven clean 删除target目录下所有生成的文件，再运用maven install将项目构建输出到本地仓库中，最后使用maven package对项目进行打包。打包成功之后会在项目的target文件夹下生成一个jar包，如果打包方式为war则生成war包。

jar包包含了Java程序的类、资源、配置文件等，jar包通常用于打包Java程序的库或应用程序。而war包中包含了Web应用程序的所有内容，如JSP、Servlet、HTML、CSS、JavaScript、图片、配置文件等。war包通常用于部署Web应用程序到Web容器（如Tomcat）中。

在项目pom文件中选择要打什么样的包，排除SpringBoot中内嵌的tomcat，使其在测试的时候不生效。

方法二：

直接在服务器上打包。首先要确定自己的Linux服务器是否配置了maven、git的环境，没有可以查询有了这些环境之后就可以进行接下来的操作了。创建文件夹用来存储代码，使用git clone 命令将代码拉取到当前文件夹下，输入git的账号和密码。进入代码文件夹下输入mvn clean清除，在输入mvn package命令打包，成功之后在项目target目录下生成jar包。

2、war包部署

首先使用kill -9 “PID”将用jar包启动的项目停止，然后将打好的war包放入Tomcat的webapps目录下面。进入到bin目录下输入./startup.sh命令启动tomcat，启动成功后，在浏览器中输入ip+8080端口号，出现tomcat的猫页面为成功。如果想要访问项目则需要在路径后面添加pom文件中artifactId标签的内容。

三、前端部署

1、上传文件
